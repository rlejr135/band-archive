# Band Archive 구현 계획서

## 1. 프로젝트 개요
밴드 연습곡, 악보, 가사 및 메모를 효율적으로 관리하기 위한 웹 애플리케이션입니다.

## 2. 현재 상태
- **Back-end**: Flask, SQLAlchemy 기반 API 구성 완료. 기본 `Song` 모델 및 `/songs` (GET, POST) 엔드포인트 구현됨.
- **Front-end**: Vite + React 기본 템플릿 프로젝트 생성됨. 아직 API 연동 및 UI 구성은 안 된 상태.

## 3. 백엔드(Backend) 개발 계획
- [ ] **Song 모델 확장**: 장르, 난이도 등 필드 추가 고려.
- [ ] **CRUD 완성**: PUT (수정), DELETE (삭제) 엔드포인트 추가.
- [ ] **파일 업로드**: 악보 이미지 또는 PDF 업로드 기능.
- [ ] **DB 마이그레이션**: Flask-Migrate 도입하여 스키마 변경 관리.
- [ ] **검색 기능**: 제목, 아티스트별 검색 API 추가.

## 4. 프론트엔드(Frontend) 개발 계획
- [ ] **디자인 시스템 구축**: TailwindCSS 또는 Vanilla CSS를 사용한 모던한 UI 디자인.
- [ ] **반응형 레이아웃**: 모바일에서도 편하게 볼 수 있는 레이아웃 구현.
- [ ] **상태 관리**: Context API 또는 Zustand를 사용한 전역 상태 관리.
- [ ] **컴포넌트 개발**:
  - 곡 목록(Song List)
  - 곡 상세(Song Detail)
  - 곡 추가/편집 폼(Form)
  - 검색창(Search Bar)
- [ ] **API 연동**: Axios 또는 Fetch API를 사용하여 백엔드와 통신.

## 5. 추가 예정 기능 (Phase 2)
- [ ] **인증/인가**: 팀원간 공유를 위한 로그인 기능.
- [ ] **멀티미디어**: 유튜브 링크 임베드, MP3 미리보기.
- [ ] **오프라인 지원**: PWA 적용하여 인터넷 없이도 악보 확인 가능하게 함.
